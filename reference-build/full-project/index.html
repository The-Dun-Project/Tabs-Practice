14-Day Project

Tab Component

All phases. All concepts. One component.

Phase 1 · Days 1–5

Foundation — Static tabs, sliding indicator, fade

Foundation

The craft of building things

Every interface you have ever used was written line by line by someone who once knew nothing. The only difference between you and them is the number of hours spent typing, breaking things, and figuring out why.

Motion

Motion earns its place

Animation is not decoration. It is information — it tells the user where they are, where something came from, and where it went. A good animation makes the interface feel alive. A bad one just makes people wait.

Transition

The fade is a conversation

When content fades out before new content fades in, you are telling the user the old thing is leaving and something new is arriving. That small moment of acknowledgment is what makes a UI feel considered.

Phase 2 · Days 6–10

Transitions — Directional slide, height, keyboard

Days 6–7 · Directional Slide

Panel One

Directional awareness

Click the tabs in order, then in reverse. Watch how content slides in the direction of your navigation. The interface tells you where you are going.

Panel Two

Left or right?

JavaScript compares the index of the clicked tab to the active tab. If the new index is higher, you moved right. Lower means left. The content enters from the same direction.

Panel Three

Direction is information

You are now managing not just WHAT is active but the relationship between the previous state and the next one. This is a big conceptual step forward.

Day 8 · Animated Height

Short content

Small panel

This panel has very little content. Switch to the Tall tab and watch the container grow smoothly.

Medium content

A bit more here

This panel has a moderate amount of content. It is taller than Short but shorter than Tall. Watch the container resize as you switch between tabs — it grows and shrinks without jumping.

Tall content

Lots of content in this one

This panel is significantly taller than the others. Without height animation, the container would snap instantly to its new size. With animation, it grows naturally — like a physical thing expanding.

JavaScript reads this panel's scrollHeight — its natural full height — and sets that as an explicit pixel value on the container. CSS transition does the rest.

This is one of the trickier animation patterns because height: auto cannot be transitioned directly. You always need an explicit pixel value to animate from and to.

Days 9–10 · Keyboard Navigation

Keyboard · Alpha

Click me then press arrow keys

Click any tab in this section. Then press your left and right arrow keys. You can navigate all three tabs without touching the mouse. This is how accessible interfaces work.

Keyboard · Beta

You pressed an arrow key

JavaScript listened for the keydown event on the tab list. It checked event.key — if ArrowRight or ArrowLeft, it calculated the next index, called .focus() and .click() on that button.

Keyboard · Gamma

It wraps around

Right on the last tab wraps to the first. Left on the first tab wraps to the last. Users should never get stuck at the edge of a component.

Phase 3 · Days 11–14

Polish — Easing, variants, mastery

Days 11–12 · Easing Playground
Easing:

Easing · Standard

Feel the difference

Select a curve above, then click between these tabs. Standard cubic-bezier feels most natural — it accelerates fast and decelerates to rest, like a physical object moving through space.

Easing · Ease Out

Snappy and decisive

Ease-out starts fast and slows to rest. It feels snappy and confident. Good for UI elements that need to feel immediately responsive. The start is instant; the landing is smooth.

Easing · Linear

Constant, mechanical

Linear moves at constant speed. No acceleration. No deceleration. It feels robotic — like a machine, not a physical object. Almost nothing in real life moves this way.

Day 14 · Variant — Pill Tabs

Variant · Pill

Same logic, new skin

The JavaScript for this component is identical to the main tabs. The only difference is the CSS. The indicator is now a filled background pill instead of a line below the text.

Variant · Pill

One pattern, many forms

Once you understand the core tab pattern — class toggling, indicator movement, panel switching — you can build any visual variant. The logic does not change. Only the appearance does.

Variant · Pill

Internalization complete

By Day 14 you should be able to close every file and rebuild any variant of this component from memory. Not because you memorized the code — but because you understand it.

Day 14 · Variant — Vertical Tabs

Vertical · Overview

Vertical layout

The tabs are on the left. The content is on the right. The JavaScript is identical to every other tab component on this page. The layout is handled entirely by CSS flexbox with flex-direction: row.

Vertical · Details

CSS does the heavy lifting

Changing from horizontal to vertical requires almost no JavaScript changes. You change the flex-direction, adjust a few borders, and the layout shifts completely. The logic stays the same.

Vertical · Notes

Layout is a design decision

Vertical tabs suit settings panels and dashboards. Horizontal tabs suit content switching in constrained spaces. Same component. Different context. Same JavaScript.
